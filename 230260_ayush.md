## 1. Introduction / Overview
The objective of this assignment is to develop a command-line tool capable of building a word-level frequency index over large text files. The system must be memory-efficient, using a fixed-size buffer for file I/O, and supports multiple file versions. Users can query the word count, compare differences between versions, and retrieve the top-K most frequent words.

## 2. Design and Approach
The system follows an Object-Oriented Programming (OOP) design, prioritizing modularity and extensibility. 
- **Modularity:** Separate classes handle file reading, tokenization, index storage, and query processing.
- **Polymorphism:** Tokenization is abstracted via a base `Tokenizer` class, allowing different splitting strategies (e.g., Alphanumeric vs. Whitespace) to be swapped dynamically.
- **Function Overloading:** Used in `VersionedIndex` for flexible index building and `QueryProcessor` for handling different query types.
- **Templates:** A generic `FrequencyMap` template manages word-to-count mapping, supporting future extensibility for different key types.

## 3. Description of Classes
- `FrequencyMap<KeyType>`: A template wrapper for `std::unordered_map` that provides a clean interface for frequency counting and ranking.
- `BufferedFileReader`: Manages binary file reading into a fixed-size heap-allocated buffer.
- `Tokenizer` (Abstract): Interface for splitting text chunks into tokens.
- `AlphanumericTokenizer`: Extracts contiguous alphanumeric tokens and converts them to lowercase.
- `WhitespaceTokenizer`: Splits by whitespace and then cleans non-alphanumeric characters.
- `VersionedIndex`: Stores and manages `FrequencyMap` instances for different named versions.
- `QueryProcessor`: Orchestrates the flow, invoking the correct logic based on CLI arguments.
- `Arguments`: A helper struct for parsing and validating command-line flags.

## 4. Fixed-Size Buffer Processing
Memory efficiency is achieved by reading the file in chunks of size B (where 256 KB ≤ B ≤ 1024 KB). 
- To avoid splitting words at buffer boundaries, the `VersionedIndex` looks for the last non-alphanumeric character in each chunk.
- Any partial word at the end of a chunk is carried over to the beginning of the next chunk (leftover processing).
- This ensures that tokens are correctly identified even if they span multiple buffer reads.

## 5. Assumptions and Observations
- **Alphanumeric Tokens:** Tokens are defined as contiguous sequences of characters [a-zA-Z0-9]. All other characters (punctuation, symbols) are treated as separators.
- **Case Insensitivity:** All words are converted to lowercase before indexing and querying.
- **Tie-breaking:** In Top-K queries, words with the same frequency are sorted alphabetically.
- **Performance:** Using `std::unordered_map` ensures average O(1) time complexity for word lookups, making the tool performant even with millions of unique tokens.
- **Platform:** The implementation targets C++17 for modern features like `std::unique_ptr` and `std::move`.
